# 第2章 词元与嵌入（Tokens & Embeddings）

在进入 Word2Vec、BERT 等稠密嵌入前，先用**词袋模型**理解最朴素的「文本 → 向量」路径。

---

## 1.2.1 词袋模型（Bag-of-Words, BoW）

### 核心定义

词袋模型是最早的文本向量化方法之一：

> 忽略语序、语法与上下文，只把句子看作「词的集合」，通过统计词频把文本转成数值向量。

- **定位**：经典的表示模型（Representation Model），是现代嵌入技术的前身。
- **背景**：可追溯到 20 世纪 50 年代；2000 年前后常用于文本分类、信息检索等任务。

### 工作流程（书中三图对应）

以两句英文为例：

- 输入 1：`That is a cute dog`
- 输入 2：`My cat is cute`

#### 步骤 1：分词（Tokenization）

- 按空格切分为 token：
  - `that, is, a, cute, dog`
  - `my, cat, is, cute`
- **局限**：仅适用于有空格分隔的语言；中文需专门分词（后续章节）。

#### 步骤 2：构建词表（Vocabulary）

- 收集不重复 token，例如：`[that, is, a, cute, dog, my, cat]`（维度 7）。
- **本质**：每个词对应向量的一维索引。

#### 步骤 3：统计词频生成向量

- 对句子 `My cat is cute`，若词表按语料中**词语首次出现顺序**排列为 `[that, is, a, cute, dog, my, cat]`，则向量为：`[0, 1, 0, 1, 0, 1, 1]`（各维为对应词出现次数）。`04_code_lab/bow_demo.py` 使用相同约定。

### 优缺点

**优点**：实现简单、可解释（一维一词）、易与传统机器学习分类器结合。  
**缺点**：丢失语序与语义；词表大时**稀疏、维度高**；停用词与实词同权。

### 技术演进（承上启下）

BoW → **TF-IDF**（缓解停用词）→ **Word2Vec**（稠密语义向量）→ **BERT / LLM**（上下文动态表示）。

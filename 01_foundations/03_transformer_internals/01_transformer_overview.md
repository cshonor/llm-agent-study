# 01 Transformer整体架构与核心思想

## 一、Transformer核心定位

Transformer 是完全基于自注意力机制、摒弃循环（RNN/LSTM）与卷积（CNN）的序列建模网络，由 Google 在 2017 年《Attention Is All You Need》中提出。它解决了 RNN/LSTM 串行计算、长依赖建模弱，以及 CNN 感受野受限的问题，是当下 LLM 与 Agent 的核心基础架构。

核心作用：将文本序列转化为语义向量（编码），并从语义向量生成文本序列（解码），支持翻译、对话、文本生成、意图识别等任务。

## 二、Transformer整体架构组成

整体分为编码器（Encoder）与解码器（Decoder）两部分：

1. 编码器：负责输入序列语义编码，输出上下文语义特征
2. 解码器：负责序列生成，基于编码器输出与已生成文本逐词生成目标文本
3. 辅助模块：Embedding、Positional Encoding、多头注意力、FFN、线性层 + Softmax

### 架构流程（对话/翻译）

1. 输入文本 -> 嵌入 + 位置编码 -> 编码器堆叠层 -> 全局语义向量
2. 编码器输出 -> 解码器交叉注意力层
3. 解码器输入（起始符/已生成文本）-> 嵌入 + 位置编码 -> 解码器堆叠层 -> 线性层 -> Softmax -> 下一个词
4. 工具/API调用场景可仅使用编码器输出语义向量

## 三、Transformer vs RNN/LSTM vs CNN

| 特性 | Transformer | RNN/LSTM | CNN |
|------|-------------|----------|-----|
| 计算方式 | 并行计算（所有词同时输入） | 串行计算（逐词） | 局部并行，卷积核滑动 |
| 长序列依赖 | 全局注意力直接建模 | 易梯度消失/爆炸 | 依赖堆叠扩大感受野 |
| 语义提取 | 全局上下文语义 | 时序语义 | 局部 n-gram 语义 |
| 适用场景 | 长文本、大模型、对话、翻译 | 简单时序任务 | 局部特征提取 |

## 四、注意力机制基本原理

### 1) 注意力核心逻辑

处理当前词时，动态关注句中其他词，权重越大表示相关性越高。

`Attention(Q,K,V) = softmax((QK^T)/sqrt(d_k))V`

- Q：当前词查询向量
- K：全体词键向量
- V：全体词值向量
- `sqrt(d_k)`：缩放，避免点积过大导致 Softmax 饱和

### 2) 多头注意力

将 Q/K/V 拆分到多个子空间并行计算，再拼接，能够同时捕捉语法、语义、指代等不同维度关联。

### 3) 位置编码

由于 Transformer 无循环结构，需显式加入位置信息，让模型理解词序。

## 五、核心总结

1. Transformer = 编码器（语义编码）+ 解码器（文本生成）
2. 并行计算与全局依赖建模能力显著优于传统 RNN/LSTM
3. 注意力机制是核心能力来源
4. 工程上可拆分：编码器做分类/向量检索，解码器做生成
